git命令笔记（1）

发表于 2022-07-11 | 分类于 git

前提：需要一个已创建好的仓库地址 例如：git@xxx.yyy.zzz

  1. 本地新增仓库名对应远程的仓库地址：
    git remote add main git@xxx.yyy.zz
    main：命名本地的仓库别名

  2. 创建本地分支：
    git checkout -b dev
    dev：创建一个名字为dev的分支

  3. 创建远程分支：
    git push main dev
    main：远程仓库名字
    dev：远程分支名字

  4. 提交文件：
    创建一个空文件
    touch test.txt
    将文件绑定git
    git add .
    提交文件至本地分支，并且备注test
    git commit -m test
    推送至远程仓库
    git push

提示报错：fatal: 没有配置推送目标。
或通过命令行指定 URL，或用下面命令配置一个远程仓库的话
执行命令：
git push –set-upstream main dev
意思是设置当前本地仓库分支对应提交到的远程仓库main的远程分支dev，以后直接执行git push就可以了

  1. 切换分支：
    假设举例的分支test存在,并且当前分支为dev
    git checkout test 切换分支为test
  2. 合并其他分支（合并dev分支至当前test分支）：
    git merge dev 该命令执行后会默认推送至远程仓库
    git merge dev –no-commit 该命令执行后不会推送至远程仓库
